מחלקה 1: Cipher (cipher.py)
================================================================================

שם המחלקה: Cipher

▪ תפקיד המחלקה
  מחלקת Cipher טוענת בכל פעולות הצפנה/פענוח AES תוך שימוש ב-EAX mode,
  וניהול חילופי מפתחות Diffie-Hellman לתקשורת מאובטחת בין client לשרת.
  היא מספקת הצפנה ברמת ה-instance (עם מפתחות משותפים) וגם שיטות סטטיות
  לחילופי מפתחות.

▪ תכונות המחלקה
  ● key
    - שמור את מפתח ההצפנה הסימטרי
    - משמש לשיטות aes_encrypt ו-aes_decrypt להצפנה/פענוח נתונים
  
  ● nonce
    - שמור את ה-nonce עבור AES-EAX mode
    - נדרש על ידי AES cipher כדי להבטיח ייחודיות בהצפנה

▪ פעולות המחלקה

  ● __init__(key, nonce)
    - key, nonce
    - אתחול אובייקט Cipher עם פרמטרי הצפנה
    - קובע cipher עם ה-key ו-nonce המסופקים לפעולות הצפנה/פענוח

  ● aes_encrypt(txt)
    - txt - טקסט רגיל להצפנה
    - החזר טקסט מוצפן
    - מצפין בתים באמצעות AES-256 ב-EAX mode; יוצר אובייקט cipher חדש
      לכל פעולה כדי להבטיח ביטחון

  ● aes_decrypt(cipher_text)
    - cipher_text - נתונים מוצפנים לפענוח
    - החזר טקסט רגיל
    - מפענח טקסט מוצפן באמצעות AES-256 ב-EAX mode עם ה-key ו-nonce
      המאוחסנים

  ● get_dh_public_key() [STATIC]
    - אין
    - החזר tuple
    - יוצר זוג מפתחות Diffie-Hellman חדש באמצעות group 14 עם
      חוזק מפתח 540-bit לחילופי מפתחות מאובטחים

  ● get_dh_shared_key(dh_1, pk_2, lngth=32) [STATIC]
    - dh_1, pk_2, lngth
    - החזר shared secret key באורך המצוין
    - חישוב shared secret מ-DH object שלו ו-public key של העמית,
      החזרת רק ה-bytes הראשונים 'lngth'

מחלקה 2: HashPasswords (hashing.py)
================================================================================

שם המחלקה: HashPasswords

▪ תפקיד המחלקה
  מחלקת HashPasswords טוענת בגיבוב וזיהוי סיסמאות באמצעות PBKDF2
  עם SHA-256 algorithm. היא מספקת אחסון סיסמאות מאובטח על ידי יצירת
  מלחים אקראיים והשוואת גיבובים לאימות.

▪ תכונות המחלקה
  (למחלקה זו אין instance attributes; כל השיטות הן instance methods ללא
   state persistence)

▪ פעולות המחלקה

  ● hash_new_password(password)
    - password - סיסמה רגילה לגיבוב
    - החזר tuple
    - יוצר 16-byte salt אקראי וחישוב PBKDF2-HMAC-SHA256
      עם 100,000 iterations; החזרת שניהם לאחסון ב-database

  ● is_correct_password(salt, pw_hash, password)
    - salt, pw_hash, password - סיסמה שסופקה על ידי המשתמש
    - החזר נכון/לא נכון
    - גיבוב הסיסמה המסופקת עם ה-salt השמור תוך שימוש באותו
      algorithm ומספר iterations, ואחר כך משתמש בקבוצה בזמן קבוע
      (hmac.compare_digest) כדי למנוע timing attacks
