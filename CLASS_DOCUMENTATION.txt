CLASS 1: Cipher (cipher.py)
================================================================================

Class Name (English): Cipher

▪ Class Role
  The Cipher class handles all AES encryption/decryption operations using the EAX
  mode, and manages Diffie-Hellman key exchange for secure communication between
  client and server. It provides both instance-level encryption (with shared keys)
  and static methods for key exchange.

▪ Class Attributes
  ● key
    - Role: Stores the symmetric encryption key (32 bytes)
    - Usage: Used by aes_encrypt and aes_decrypt methods to encrypt/decrypt data
  
  ● nonce
    - Role: Stores the nonce (number used once) for AES-EAX mode
    - Usage: Required by AES cipher to ensure uniqueness in encryption

▪ Class Operations

  ● __init__(key, nonce)
    - Input Arguments: key (bytes), nonce (bytes)
    - Output: Initializes the Cipher object with encryption parameters
    - Functionality: Sets up the cipher with the provided key and nonce for future
      encryption/decryption operations

  ● aes_encrypt(txt)
    - Input Arguments: txt (bytes) - plaintext data to encrypt
    - Output: Returns encrypted ciphertext (bytes)
    - Functionality: Encrypts the input bytes using AES-256 EAX mode; creates a new
      cipher object for each operation to ensure security

  ● aes_decrypt(cipher_text)
    - Input Arguments: cipher_text (bytes) - encrypted data to decrypt
    - Output: Returns decrypted plaintext (bytes)
    - Functionality: Decrypts the input ciphertext using AES-256 EAX mode with the
      stored key and nonce

  ● get_dh_public_key() [STATIC]
    - Input Arguments: None
    - Output: Returns tuple (DiffieHellman object, public key bytes)
    - Functionality: Generates a new Diffie-Hellman key pair using group 14 with
      540-bit key strength for secure key exchange

  ● get_dh_shared_key(dh_1, pk_2, lngth=32) [STATIC]
    - Input Arguments: dh_1 (DiffieHellman object), pk_2 (bytes - peer's public key),
      lngth (int - desired key length, default 32)
    - Output: Returns shared secret key (bytes) of specified length
    - Functionality: Computes the shared secret from own DH object and peer's public
      key, returning only the first 'lngth' bytes

CLASS 2: HashPasswords (hashing.py)
================================================================================

Class Name (English): HashPasswords

▪ Class Role
  The HashPasswords class handles password hashing and verification using PBKDF2
  with SHA-256 algorithm. It provides secure password storage by generating random
  salts and comparing hashes for authentication.

▪ Class Attributes
  (This class has no instance attributes; all methods are instance methods without
   state persistence)

▪ Class Operations

  ● hash_new_password(password)
    - Input Arguments: password (str) - plaintext password to hash
    - Output: Returns tuple (salt bytes, pw_hash bytes)
    - Functionality: Generates a random 16-byte salt and computes PBKDF2-HMAC-SHA256
      hash with 100,000 iterations; returns both for storage in the database

  ● is_correct_password(salt, pw_hash, password)
    - Input Arguments: salt (bytes), pw_hash (bytes - stored hash), 
      password (str - user-provided password)
    - Output: Returns boolean (True if password matches, False otherwise)
    - Functionality: Hashes the provided password with the stored salt using same
      algorithm and iteration count, then uses constant-time comparison (hmac.compare_digest)
      to prevent timing attacks
